# 動画クオリティ・チェッカー

YouTube動画の最初の180秒を1秒ごとに解析し、収益化審査の合格水準かを5段階で判定する無料Webツール。

**公開URL**：https://shinji-design.github.io/yt-quality-checker/

## 機能

- 動画の最初の3分（180秒×1秒刻み）を解析
- 5項目スコア（動きのバランス／止まっている時間／場面切替／動きの種類／集中度）
- 5段階判定（合格水準・ほぼ合格・もうすぐ合格・要改善・大幅作り直しが必要）
- 優先度付きの具体的な改善アクション（最大3件）

## 仕組み

ブックマークレット方式：

1. ユーザーが GitHub Pages 上の本ツールページを開き、ブックマークレットをブラウザに登録
2. YouTube 動画ページでブックマークレットをクリック
3. ブックマークレットが `<video>` 要素を1秒ずつシークし、80x45 canvas にフレームを描画してピクセル差を計算
4. 180秒分の差分データを base64 エンコードした URL ハッシュで本ツールに戻す
5. 本ツールがスコアリング・判定・アドバイス生成・描画

**バックエンド不要・サーバ不要**。ユーザー自身のブラウザのIPで処理されるため、データセンターIPの bot 検出問題は発生しません。

## ローカルで動かす

任意のHTTPサーバで `docs/` ディレクトリを配信するだけ。

```bash
cd docs
python3 -m http.server 8000
# http://localhost:8000 で開く
```

注意：ブックマークレットは生成時の URL を内部に埋め込むため、ローカルで生成したブックマークレットは本番では動きません（デプロイ後の本番URLで取り直してください）。

## デプロイ（GitHub Pages）

1. リポジトリの Settings → Pages
2. Source: Deploy from a branch
3. Branch: `main` / `/docs`
4. Save
5. 数分で `https://<USERNAME>.github.io/<REPO>/` で公開される

## 判定アルゴリズム

180秒分の連続フレーム間 RGB ピクセル差 (0–255 スケール) から以下を計算：

| 指標 | 意味 |
|---|---|
| mean | 平均ピクセル差 |
| sharpCut | ≥60 の比率（急激なカット率） |
| lowChange | <20 の比率（ほぼ静止の比率） |
| eventRate | ≥20 の比率（場面切替頻度） |
| maxBand | 最も多い差分帯の占有率 |
| usedBands | 5秒以上入った差分帯の数（8段階中） |

これらから 5 スコアを計算（0–100）し、平均で総合スコアを出す。

判定基準は実証データ（複数チャンネル552件のOK/NG判定済みデータ）から導出。
